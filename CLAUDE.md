## Architecture

This is a **Next.js 16** app using **App Router** with a **feature-based architecture**. Code is organized by business domain rather than technical layer.

### Project Structure

```
src/
├── app/                    # Next.js App Router (routes)
│   ├── (main)/            # Route group with shared layout (Header/Footer)
│   ├── api/               # API routes
│   └── globals.css
├── components/
│   ├── layout/            # Layout components (Header, Footer)
│   ├── providers/         # Provider components (Redux, etc.)
│   └── ui/
│       ├── mui/           # MUI wrapper components (use in server components)
│       └── [tailwind]     # Tailwind-based UI components
├── features/              # Feature-based modules (core pattern)
│   ├── auth/              # Authentication feature
│   ├── posts/             # Posts feature
│   ├── comments/          # Example feature using Redux factory
│   └── shared/            # Shared feature components & UI state
├── lib/                   # Utility functions
│   ├── api.ts             # fetcher wrapper
│   ├── api-base.ts        # ApiBase class for CRUD
│   ├── redux-factory/     # Generic Redux slice/hooks factory
│   └── utils.ts           # cn(), formatDate(), truncate()
├── store/                 # Global Redux store configuration
│   ├── index.ts           # Store setup, root reducer
│   └── hooks.ts           # Typed useAppDispatch, useAppSelector
├── theme/                 # MUI theme configuration
│   ├── index.ts           # Main theme with typography, shadows, z-index
│   └── palette.ts         # Colors mapped from Tailwind tokens
├── types/                 # Global type definitions
└── tailwind.config.ts     # Tailwind with sharedClasses export
```

### Feature Pattern

Each feature in `src/features/` follows this structure:

```
feature-name/
├── components/            # Feature-specific components
├── hooks/                 # Feature-specific React hooks
├── redux/                 # Redux slice (generated by factory or manual)
├── services/              # API calls / business logic
├── types/                 # Feature-specific TypeScript types
└── index.ts               # Public API exports
```

**Key pattern**: Export from `index.ts` files. Import from `@/features/feature-name/section`, not from nested paths.

## Redux + State Management

This project uses **Redux Toolkit** with a **custom factory system** to eliminate code duplication.

### Redux Factory System (IMPORTANT)

**Location**: `src/lib/redux-factory/`

The factory generates complete Redux slices and hooks for any feature with:
- CRUD operations (create, read, update, delete, deleteMany)
- Pagination (page, pageSize, totalPages, hasMore)
- Search, sort, filters
- Entity caching by ID
- Optimistic updates
- Loading states (isLoading, isCreating, isUpdating, isDeleting)
- Error handling

**Creating a new feature with the factory** (~50 lines total):

```typescript
// 1. Define types (10 lines)
// src/features/users/types/user.types.ts
interface User { id: string; name: string; email: string; }
type CreateUserInput = Omit<User, 'id'>;
type UpdateUserInput = Partial<Omit<User, 'id'>>;

// 2. Create API service (5 lines)
// src/features/users/services/users.ts
import { createApiService } from '@/lib/redux-factory';
export const usersApi = createApiService<User>('/users');

// 3. Generate Redux slice (10 lines)
// src/features/users/redux/index.ts
import { createCrudSlice } from '@/lib/redux-factory';
import { usersApi } from '../services/users';
export const usersSlice = createCrudSlice<User, CreateUserInput, UpdateUserInput>({
  name: 'users',
  apiService: usersApi,
});

// 4. Generate hooks (10 lines)
// src/features/users/hooks/index.ts
import { createCrudHooks } from '@/lib/redux-factory';
import { usersSlice } from '../redux';
export const { useList: useUsers, useEntity: useUser } = createCrudHooks({
  slice: usersSlice,
  sliceName: 'users',
});

// 5. Add reducer to store (src/store/index.ts)
// users: usersSlice.reducer,

// 6. Use in component
const { users, isLoading, create, update, remove, changePage, search } = useUsers();
```

**Factory exports**:
- `createCrudSlice<TEntity, TCreateInput, TUpdateInput>(config)` - Generates Redux slice
- `createCrudHooks(config)` - Generates hooks (useList, useEntity, useSelection)
- `createApiService<TEntity>(endpoint)` - Creates API service extending ApiBase
- `createMockApiService<TEntity>(mockData)` - Creates mock API for development

### Store Configuration

**File**: `src/store/index.ts`

The store uses Redux Toolkit with:
- Combined reducers from each feature
- Typed hooks (`useAppDispatch`, `useAppSelector`)
- Redux DevTools in development
- Serializable check disabled for Next.js compatibility

### Redux Hooks

```typescript
import { useAppDispatch, useAppSelector } from '@/store/hooks';

// Typed dispatch
const dispatch = useAppDispatch();
dispatch(fetchPosts());

// Typed selector
const posts = useAppSelector((state) => state.posts.list.data?.items);
```

## Styling

This project uses a **hybrid approach**: Tailwind CSS + Material UI (MUI).

### Tailwind CSS
- **Tailwind CSS v4** via `@tailwindcss/postcss`
- **Custom config**: `tailwind.config.ts` includes:
  - Extended color palette (primary, secondary, accent, neutral, success, warning, error)
  - Spacing, typography, shadows, animations, z-index scales
  - **`sharedClasses` export**: Pre-configured class strings for consistent styling
- **Utility**: `cn()` from `@/lib/utils` merges clsx + tailwind-merge
- **Dark mode**: Uses `class` strategy (add `dark` class to `<html>`)

### Material UI (MUI)
- **MUI v5** with Emotion styling
- **Theme**: `src/theme/` maps Tailwind colors to MUI palette
- **Wrapper components**: `src/components/ui/mui/` provides client wrappers for use in server components
- **Available wrappers**: MuiButton, MuiCard, MuiCardContent, MuiCardActions, MuiTextField, MuiTypography, MuiStack, MuiBox, MuiChip, MuiAlert

## Data Fetching & API

### ApiBase Class

**File**: `src/lib/api-base.ts`

Generic CRUD API class that all API services should extend:

```typescript
class ApiBase<T> {
  list(params): Promise<PaginatedList<T>>
  getById(id): Promise<T>
  create(data): Promise<T>
  update(id, data): Promise<T>
  delete(id): Promise<void>
  deleteMany(ids): Promise<void>
}
```

### Fetcher Utility

**File**: `src/lib/api.ts`

Simple fetch wrapper for non-CRUD calls:

```typescript
import { fetcher } from '@/lib/api';
const data = await fetcher<Post>('/api/posts', { method: 'GET' });
```

## Type Definitions

**File**: `src/types/global.ts`

Key types:
- `AsyncState<T>` - Loading state pattern (data, isLoading, error, lastUpdated)
- `PaginationParams` - Pagination parameters (page, pageSize, sortBy, sortOrder, search, filters)
- `PaginatedList<T>` - Paginated response (items, total, page, pageSize, totalPages, hasMore)
- `EntityState<T>` - Normalized entity state (byId, allIds, isLoading, error)
- `ApiResponse<T>` - Standard API response wrapper
- `ApiError` - Error shape
